## 文件命名规范

详见 `paper/docs/文件命名规范.md`

### 核心规则

1. **当前有效版本**存放在 `paper/current/`
2. **命名格式**：`v{主版本}.{次版本}_{描述}.md`
3. **最新版本**永远是版本号最高的文件
4. **自动归档**：写入新版本时，旧版本自动归档至 `paper/versions/`

### 目录结构

```
paper/
├── current/      - 当前有效版本
├── versions/     - 历史版本归档
├── drafts/       - 草稿文件
├── backups/      - 修改前备份
├── docs/         - 项目文档
├── references/   - 参考文献原始文件
├── reports/      - 验证报告
└── style-references/ - 写作风格参考论文
```

### MCP写作风格参考功能

#### 功能说明

在使用 `paper_writer` 工具撰写论文章节时，系统会自动：

1. 检测 `paper/style-references/` 目录中的PDF文件
2. 提取PDF文本内容
3. 分析写作风格特征（句式模式、过渡词使用、段落组织方式等）
4. 将风格特征应用到生成的内容中

#### 如何添加参考论文

1. 将PDF文件复制到 `paper/style-references/` 目录
2. 下次使用 `paper_writer` 时会自动检测并应用

#### 分析的风格特征

| 风格要素 | 具体表现 |
|----------|----------|
| 总结句式 | "在XX方面"，而非"针对问题X" |
| 过渡方式 | "围绕XX""总体来看""近年来"等简洁过渡 |
| 段落组织 | 先总述→分述→总结，层层递进 |
| 论述逻辑 | "问题→方法→成果→意义"为线索 |
| 局限性表述 | "受XX限制，仍存在XX局限" |
