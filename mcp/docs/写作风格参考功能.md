# 写作风格参考功能

## 概述

本功能允许用户提供参考论文PDF文件，系统会自动分析其写作风格特征（如句式模式、过渡词使用、段落组织方式等），并在撰写论文时应用这些风格特征，使生成的内容更符合用户期望的学术写作风格。

## 使用方法

### 1. 放置参考论文

将参考论文的PDF文件放入以下目录：

```
paper/style-references/
```

如果目录不存在，可手动创建：

```bash
mkdir -p paper/style-references
```

### 2. 写作时自动应用

在使用 `paper_writer` 工具撰写论文章节时，系统会自动：

1. 检测 `paper/style-references/` 目录中的PDF文件
2. 提取PDF文本内容
3. 分析写作风格特征
4. 将风格特征应用到生成的内容中

### 3. 查看风格参考状态

写作完成后，输出信息会显示：

```
**写作风格参考**：总结句式X种，常用过渡词X个，段落开头偏好：XX式
```

如果没有找到参考论文，会显示提示：

```
💡 **提示**：未找到写作风格参考论文。如需参考特定论文的写作风格，请将PDF文件放入 paper/style-references/ 目录。
```

## 分析的风格特征

### 1. 句式特征
- **总结句式**：如"在XX方面，本文提出了..."
- **过渡句式**：如"围绕上述问题..."
- **局限性表述**：如"受XX限制，本研究仍存在..."

### 2. 段落组织
- **开头方式**：问题导向式、范围界定式、总结式、直接陈述式
- **展开方式**：递进式、并列式、对比式
- **结尾方式**：总结展望式

### 3. 词汇偏好
- **正式学术用语**：如"提出""构建""归纳"等
- **过渡词使用频率**：如"围绕""总体来看""然而"等
- **模糊限制语**：如"一定程度上""较为""相对"等

### 4. 论述逻辑
- **问题陈述方式**：直接陈述式、问题导向式、对比引出式
- **证据呈现方式**：案例支撑式、案例归纳式、实证支撑式
- **结论推导方式**：归纳总结式、综合归纳式、证据推导式

## 技术实现

### 模块结构

```
mcp/src/services/style-analyzer.js    # 风格分析服务
mcp/src/tools/paper-writer.js         # 集成风格分析到写作流程
```

### 依赖

- Python + pdfplumber（PDF文本提取）
- Node.js（主程序）

### 分析流程

```
1. 检测参考论文PDF
   ↓
2. 提取PDF文本（pdfplumber）
   ↓
3. 正则匹配分析风格特征
   ↓
4. 生成风格应用指导
   ↓
5. 应用到LLM提示词
   ↓
6. 生成符合风格的论文内容
```

## 注意事项

1. **参考论文数量**：建议提供1-3篇风格相似的论文作为参考
2. **PDF格式**：需为可提取文本的PDF（非扫描图片）
3. **风格匹配**：系统主要分析中文论文的写作风格
4. **降级处理**：如果风格分析失败，将使用默认学术写作规范
